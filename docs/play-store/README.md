# 집중행 — 플레이 스토어 출시 준비

이 폴더는 Google Play 출시에 필요한 자료를 모아둔 곳입니다.

## ✅ 기술 준비 (완료)
- **서명**: 업로드 키스토어 생성 + `build.gradle.kts` release 서명 연결 완료.
- **AAB 빌드**: `flutter build appbundle --release`
  → `build/app/outputs/bundle/release/app-release.aab` (릴리스 키로 서명됨, `jarsigner -verify` 통과)
- **타깃**: `targetSdk 36` (Android 16) · `minSdk 24` · `versionCode 1` / `versionName 1.0.0`
- **권한**: `INTERNET`(지도 타일), `POST_NOTIFICATIONS`(진행 알림)
- **앱 이름**: 집중행 · **applicationId**: `com.gusle01.korean_focus`

### 새 버전 낼 때
`pubspec.yaml`의 `version:`을 올린다 → 예 `1.0.0+1` → `1.0.1+2`
(`+` 뒤 숫자 = versionCode, 매 업로드마다 반드시 증가). 그 후 다시 `flutter build appbundle --release`.

## 🔐 업로드 키스토어 (매우 중요 — 백업 필수)
- 파일: `android/upload-keystore.jks` — **git에 올라가지 않음**(`.gitignore` 처리). 별칭 `upload`.
- 비밀번호: `android/key.properties`(역시 git 미추적, 로컬에만 존재).
- ⚠️ **이 키스토어를 잃어버리면 같은 키로 앱을 업데이트할 수 없습니다.** 안전한 곳(클라우드/외장디스크)에 `upload-keystore.jks`와 비밀번호를 반드시 백업하세요.
- Google **Play 앱 서명**을 사용하면(권장), 이 키는 "업로드 키"가 되고 분실 시 재설정이 가능합니다.
- 지문 확인: `keytool -list -v -keystore android/upload-keystore.jks`

## 🖼️ 그래픽 에셋 (`assets/`)
- `icon-512.png` — 고해상도 아이콘 512×512 (필수)
- `feature-graphic-1024x500.png` — 피처 그래픽 (필수)
- `screenshots/` — 폰 스크린샷 후보 3장 (최소 2장 필요)
  - 출시 전 깨끗한 상태(상태바 정리)로 다시 캡처하는 걸 권장.

## 🔒 개인정보처리방침
- 초안: [`privacy-policy.md`](privacy-policy.md) — **연락 이메일을 채운 뒤** 공개 URL로 호스팅 필요
  (예: GitHub Pages, Google 사이트, Notion 공개 페이지).
- 호스팅한 URL을 Play Console "개인정보처리방침"에 입력.

## 📋 Play Console 단계 (수동)
1. **개발자 계정** 등록(최초 1회 $25).
2. **앱 만들기**: 이름 `집중행`, 기본 언어 한국어, 앱/게임=앱, 유료/무료=무료.
3. **앱 서명**: Play 앱 서명 사용(권장).
4. **프로덕션(또는 비공개 테스트) 트랙 → 새 릴리스**: `app-release.aab` 업로드.
5. **스토어 등록정보**: 위 설명 + `icon-512.png` + `feature-graphic-1024x500.png` + 스크린샷(2장 이상).
6. **개인정보처리방침 URL** 입력.
7. **앱 콘텐츠**: 콘텐츠 등급 설문 / **데이터 안전**(수집·공유 없음, 로컬 저장만) / 타겟 연령 / 광고 없음 / 정부앱 아님.
8. **국가·지역**, 가격(무료) 설정.
9. **검토 제출** → 승인까지 보통 수일.
