# 집중행 — 추후 패치 예정 (Upcoming Patches)

향후 적용 예정인 기능/변경의 계획을 모아두는 문서. 확정·구현되면 이 문서에서 내리고 커밋/CHANGELOG로 옮긴다.

## [예정] 환승 여정 (멀티구간) — Multi-leg Journey

- 상태: **계획 정리됨 / 구현 대기**
- 작업량: **큼** (2단계 제안)
- 결정 사항: A~E (아래) 확정 후 1단계 착수

### 1. 개념
한 번의 집중을 **여러 정차역을 거치는 장거리 여정**으로 확장한다.
예: `서울 →〔1구간 집중〕→ 대전 정차·휴식 →〔2구간 집중〕→ 부산 도착`.
각 구간은 지금의 단일 여정과 동일하게 동작(타이머·지도·실시간 현황·도착)하고,
사이에 **정차(휴식 + 탑승)** 가 들어간다 = 한국형 뽀모도로(정차역에서 쉬어가기).

### 2. 사용자 흐름 (MVP)
1. 홈 → 교통수단·출발 선택(현행 동일)
2. 여정 확인 화면에 **"정차역 추가"**: 출발→도착 사이에 경유역을 순서대로 추가(2~3정차). 구간마다 집중 길이 지정.
3. "출발" → **1구간** 집중(타이머/지도/실시간 현황)
4. 1구간 도착 → **정차 화면**: "대전 정차 · 휴식 N분" + `[다음 구간 탑승]`(휴식 스킵 가능)
5. 탑승 → **2구간** 집중 … 반복
6. 마지막 구간 도착 → **최종 도착 화면**(특산품 보상 + 여정 요약: 총 구간/거리/시간)

### 3. 구조 변경 (요지)
- **새 모델**: `JourneyLeg{transport, origin, dest, plannedMinutes}` + `TripPlan{List<JourneyLeg>, currentIndex}`
- **새 상태**: `tripProvider`(전체 구간 + 현재 인덱스). 현재 구간을 기존 타이머/선택 로직에 그대로 먹여
  **검증된 per-leg 기계(타이머·실시간현황·보상·세션기록)를 거의 그대로 재사용** → 리스크 최소화
- **여정 빌더**: 확인 화면에 정차역 누적 UI(기존 `/place` 재사용)
- **새 `/transfer` 화면**: 정차 안내 + 짧은 휴식 카운트다운(스킵 가능) + 탑승
- **focus 화면**: 구간 끝 → 다음 구간 있으면 `/transfer`, 없으면 `/complete`
- **저장(앱 종료 대비)**: `ActiveJourneyRepository`를 trip(구간 리스트 + 현재 인덱스 + 현재 구간 타이머)으로 확장
  → 죽였다 켜도 현재 구간으로 복원
- **실시간 현황(LA/알림)**: 경로 = 현재 구간 "출발 → 도착" + `(2/3구간)`. 기존 갱신 경로 재사용
- **기록/통계**: **구간마다 `FocusSession` 1건** 기록 → 스트릭·통계·거리·업적·진열장 로직 **무수정 재사용**
  (같은 trip은 tripId로 묶어 표시는 2단계)

### 4. 지도
- **MVP: 현재 구간만** 표시(현행 painter 그대로 — 변경 최소)
- 2단계: 전체 다구간 경로 + 현재 구간 하이라이트

### 5. 결정할 것 (추천 ✅)
| | 항목 | 추천 |
|---|---|---|
| A | 구간 길이 기본값 | 구간별 실제 소요시간(있으면)/없으면 프리셋 (현행 규칙) |
| B | 정차 휴식 | 짧은 휴식 카운트다운 + "탑승", 스킵 가능 |
| C | 보상 | 최종 도착=특산품, 정차역=가벼운 "정차 도장" (구간마다 특산품은 인플레) |
| D | 최대 구간 | MVP 3구간까지 |
| E | 교통수단 | MVP는 여정 전체 동일 수단(다정차, 예: KTX 서울–대전–부산). 혼합 환승(기차→버스)은 2단계 — 정차역이 역↔터미널로 갈려 복잡 |

### 6. 단계
- **1단계(MVP)**: 위 흐름 전체 동작(다정차, 정차 화면, per-leg 기록, 실시간현황, 저장 복원). 지도는 현재 구간만.
- **2단계**: 다구간 지도, 환승 업적, 히스토리 trip 그룹, 휴식 연출 강화.
